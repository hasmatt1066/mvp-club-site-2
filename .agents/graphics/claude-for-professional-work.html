What "context" actually means

Two ingredients. One goes in the instructions. One lives in your real tools. Claude needs both.

Ingredient 1
How-to guides
The onboarding doc you'd hand a new hire. How you do this task, in what order, in what voice.
e.g. "Always start KB articles with a one-sentence definition."
+
Ingredient 2
Reference material
The specific, constantly-changing stuff about your actual work. Clients, numbers, meeting notes, open threads.
e.g. the Google Drive folder where meeting transcripts live.
=
What Claude needs
Context
Enough to do the task the way you'd do it, without you copy-pasting every time.

Graphic 4 · Placement: opening the "The iteration loop" section

The iteration loop

Every time Claude gets it "not quite right," the fix is thirty seconds upstream, not in the output.

1
Write instructions
A two-paragraph onboarding brief. The first draft will be incomplete. That's fine.
2
Run a real task
Actual work, not a test prompt. You learn more from one real run than ten practice ones.
3
Notice the "ugh"
Wrong tone, skipped step, phrase you hate. Stop. Don't just fix the output.
4
Add one sentence
Back to the instructions. One sentence that would have prevented it. Save.
↩ Next run is better. Do this ten times over a month.
